PHP and tumblr

06-17-2015

Sometimes, when I read some articles about PHP (if you can call them articles lol), I stumble upon interesting PHP blogs on tumblr. There's variety of PHP blogs which take different side of spectrum, either embracing PHP, or discouraging from PHP. There's also blogs that make fun of PHP. Honestly, I like some of those jokes, haha.

This post is about the last type of PHP blogs on tumblr, about funny blogs.

PHP: The Good Parts

Our first candidate for PHP blog of the year is PHP: The Good Parts.

At first sight, I didn't get the whole point, but then... after few minutes staring at the only blog post it has the "No posts yet" post, I get the idea. I was like literally rolling on the floor, you should totally check all of its posts lol.

That's very interesting, how hilarious is combination of community state and a simple phrase "No posts yet" in the blog about PHP's Good Parts can create such comical effect, "such wow" the only thing I can say.

No PHP

The second candidate is ... pam, pam, pam ... No PHP. This blog is reaction gif type blog, there's not much of gifs yet, but they're hilarious!

Some of these gifs are outdated: the one about PHP documentation, and about PHP community, but who cares (unless you're offended by these gifs)? They're actually funny.

BONUS: php sadness is like No PHP, but without funny reaction gifs, however, there's more detailed explanation why specific PHP features sucks.

PHP Manual Masterpieces

The last and the third release candidate is PHP Manual Masterpieces. This blog is a collection of PHP official documentation flaws. Some of these flaws are really funny, especially this one:

DateTimeImmutable::modify()

Alters the timestamp. Like DateTime::modify() but works with DateTimeImmutable.

Duh. I don’t see what’s so hard about this, @pda.

Of course, it doesn’t actually do what the name says and modify an unmodifiable object at all. So, at least there’s that; PHP didn’t actually go through with its threat.

All I can say, those blogs are entertaining indeed.
